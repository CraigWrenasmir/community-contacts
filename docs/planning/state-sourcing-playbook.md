# State Sourcing Playbook

This project should be run as a regional sourcing operation, not as a random statewide web search.

## Operating principle

- Discovery sources and verification sources are different.
- Discovery sources help find organisations quickly.
- Verification sources are the organisation's own public website or official council/government page.
- No record should be added to the live database unless the final contact details have been verified from an official public source.

## What a "good" lead source looks like

- Local council community directories
- Council library directories
- Community hub directories
- Neighbourhood house networks
- Primary Health Network service finders
- Multicultural or settlement service directories
- Youth and family service directories
- Regional disability/community directories
- Local community radio or event/community listings

## What to avoid as direct import sources

- Commercial directories
- Aggregator sites with unclear update ownership
- Stale PDFs unless used only as a lead source
- Pages listing personal staff contacts instead of public organisation contacts
- Social media pages without a stable organisation website

## Repeatable workflow

1. Split the state or territory into practical sourcing regions.
2. Find 3-8 high-trust discovery sources for each region.
3. Extract likely organisations by category from those sources.
4. Open each organisation's official page.
5. Verify public email, phone, website, suburb/postcode, and broad organisation fit.
6. Add the record only if it meets the minimum standard.
7. Run a gap analysis by category and geography.
8. Do a final quality pass on weak regions and weak categories.

## Recommended regional split

Each state should be broken into:

- capital city / metro
- major surrounding metro belt
- large regional centres
- secondary regional areas
- remote / far regional areas where relevant

Example for NSW:

- Sydney metro
- Hunter
- Central Coast
- Illawarra / Shoalhaven
- North Coast
- New England / North West
- Central West / Orana
- Riverina / Murray
- South East NSW
- Far West

## Coverage targets

A state is not "healthy" just because it has a high row count.

It should aim for:

- all agreed categories represented
- no major region left empty
- at least 2-3 strong records in lighter categories where possible
- a high proportion of public organisation emails
- obvious local anchor organisations in major centres

## Evidence to keep

For every sourcing pass, keep:

- the discovery source list
- the verification source URL for each record
- the date verified
- notes where an email is missing or protected

## Quality hierarchy

Prefer:

1. generic organisation email + phone + website + clear location
2. organisation email + website + clear location
3. phone + website + clear location

Avoid:

- guessing emails
- importing unverified records
- treating third-party directories as source of truth

## What the user needs to do

Very little.

The user is most valuable as:

- feedback on whether results are useful in practice
- identification of obvious local gaps
- prioritisation of states, regions, or categories

The agent should take carriage of:

- finding regional directories
- methodically harvesting leads
- verifying records
- maintaining consistency across states
